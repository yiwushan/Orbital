# Orbital v1.0.0 发布说明

发布日期：2026-04-06  
发布分支：`master`  
发布标签：`v1.0.0`

## 1. 版本定位

`v1.0.0` 是当前可稳定日常使用的里程碑版本，重点完成了：

- OnePlus 6 适配后的 Dashboard 基础布局稳定化
- 本机 CPU / 内存 / 磁盘 / 电池 / 网络主卡片与详情交互完善
- 两台内网远端服务器面板（CPU / 内存 / 磁盘）及详情能力落地
- 远端刷新策略（常态省资源 + 点击后短时高频）可用

## 2. 核心功能（v1）

### 2.1 本机监控

- CPU：总占用、8 核心负载水位、温度显示、历史
- 内存：占用信息、历史、内存明细、进程榜（CPU/MEM/IO/NET 可切换）
- 磁盘：占用信息与分区详情
- 电池：电量状态与详情
- 网络：上下行速率与历史曲线、接口详情

### 2.2 远端监控（2 台）

- 支持配置独立 `host / port / name`
- CPU 核心按 8 组聚合显示
- 远端 CPU / MEM / DISK 卡片可单独点击查看详情
- 历史图为散点，按真实时间戳绘制（非等距伪时间）

## 3. 远端刷新策略（v1）

- 基础周期：`60` 秒/次（可配置）
- 用户点击远端卡片后：进入 `5` 秒/次的短时加速刷新
- 加速持续：`60` 秒，随后自动恢复基础周期
- 历史窗口：最近 `1` 小时

## 4. 部署与运维约定

- 代码仓库（fork）：`https://github.com/yiwushan/Orbital.git`
- 设备运行目录：`/home/athbe/Orbital`
- systemd 服务：`orbital.service`
- 编译并行策略：默认使用 `nproc/2`（降低过热风险）

详细流程见：[`docs/UPDATE_DEPLOY_SOP.md`](docs/UPDATE_DEPLOY_SOP.md)

## 5. 建议的后续版本方向（v1.x）

- 远端面板点击区域进一步可视化反馈（按压态/toast）
- 远端离线与权限错误提示分级（网络异常 / 认证异常 / 命令异常）
- 远端详情页加入更清晰的时间刻度标注（如 `-60m ~ now`）
